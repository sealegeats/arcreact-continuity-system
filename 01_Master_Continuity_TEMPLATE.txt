MASTER CONTINUITY DOCUMENT
[Project Name] | v[X.Y] | [Month Year]

Purpose: Project-level source of truth. If a teammate (or LLM) reads only one
continuity doc, this is the one. Captures mission, scope, current state,
active workstreams, and the running decisions log. Pin this first in any
LLM session; pin the other continuity docs alongside as needed.

FORMAT NOTE: Plain text (.txt). Maintained as plaintext to ensure
LLM-agnostic compatibility (Gemini, ChatGPT, Claude, Copilot, etc.) and
to prevent formatting drift between editors.

VERSION CONTROL NOTE: Version bumps require approval from the designated
Continuity Steward (see Team & Handoff Continuity doc). Sub-version bumps
(v1.0 -> v1.1) for content additions; major bumps (v1.x -> v2.0) for
structural changes to the doc itself.


================================================================================
LLM SESSION BOOTSTRAP
================================================================================

Paste the following at the start of any LLM session that needs project
context. This block is designed to bring any model up to speed in one paste.

  >>> BEGIN BOOTSTRAP <<<
  You are assisting with [Project Name], a [one-sentence project
  description]. The project is being executed by a multi-person Agile
  team. I am [your role / name optional]. Today's session focus is
  [one-line goal].
  
  Attached/pasted below are the current continuity documents. Read them
  in order: Master, then [other relevant docs for this session]. Do not
  invent context not present in these docs. If you need information that
  is not in the docs, ask before proceeding.
  
  Conventions to observe:
    - Do not modify any artifact without proposing the change first.
    - Use plain text for any continuity doc updates.
    - Flag uncertainty explicitly; do not guess at file paths, names,
      or technical decisions.
  >>> END BOOTSTRAP <<<
